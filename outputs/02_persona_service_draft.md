## パターン1: B主軸 × A抱き合わせ「EC月次経営レポート自動化」

### ターゲット顧客像（1人の架空ペルソナ）

- **名前（仮）**: 田島 健一さん
- **年齢・役職**: 41歳・代表取締役
- **会社規模**: 売上6億円 / 従業員12名（うちEC運用5名、CS3名、商品開発2名、バックオフィス2名）/ 設立7年
- **業種・商材**: 自社ブランドの健康食品・サプリを楽天・Amazon・自社Shopifyの3チャネルで展開（楽天比率55%）
- **実在クライアントの近似像**: **百福堂に近い**（楽天月商800万、健康食品系、自社ブランド）
- **月次の経営課題（3つ、具体的に）**:
  1. 楽天・Amazon・Shopifyのデータがバラバラで、月次の経営判断（広告配分・在庫発注・新商品投入タイミング）が経験と勘ベース。3チャネル統合の数字を作るのに、月初に運用責任者が丸2日かかる
  2. 楽天RPP広告のROASが店舗ごとに違うが、誰も体系的に分析していない。広告費月150万のうち「効いてない30万」がどれか毎月特定できない
  3. 取締役会用の月次レポートを社外CFO（業務委託）に依頼しているが、月10万払って「Excelで作っただけのもの」が来る。経営の意思決定に使える示唆がない
- **いま使っているツール／外注先（月額付き）**:
  - 楽天RMS / Amazonセラーセントラル / Shopify（基本料月3万）
  - Looker Studio（無料、ただし誰もメンテできていない）
  - 楽天運用代行A社（月25万、運用＋簡易レポート）
  - 社外CFO（月10万、月次レポート作成）
  - Slack / Notion / freee会計
- **なぜこの人が「自動化」に金を払うか（1行）**: 自分の時間を奪っているのは「データ統合と数字作り」で、「経営判断そのもの」ではないと自覚しているから

### サービス内容（最初に売る1プロダクト）

- **商品名（仮）**: 「EC経営ダッシュボード Pro」（月次経営レポート自動配信サービス）
- **何をする**: 楽天・Amazon・Shopifyの売上/広告/在庫/CVRデータを毎日自動収集 → 月初3営業日目に「経営判断付きの月次レポート（PDF/Slides）」をSlackとメールに自動配信。週次でも要約Slack通知。常時アクセスできるダッシュボードURLも提供
- **顧客の作業（オンボーディング）**: 初回90分のヒアリング（KPI設計・閲覧者設定）+ APIキー連携（楽天RMS / Amazon SP-API / Shopify Admin API）+ 過去6ヶ月データのインポート。所要1〜2週間
- **納品物**:
  - 月次PDF/Slidesレポート（経営者向け6〜10枚、要点+示唆+次月アクション3つ）
  - 週次Slack通知（前週KPIサマリー、異常値アラート）
  - 常時アクセスのダッシュボードURL（GitHub Pages or Cloud Runで店舗別画面）
  - 月1回30分の「数字読み合わせ」オンラインMTG（オプション）
- **月額単価（仮置き）**: **月15万円〜25万円**（基本15万 / 月次読み合わせMTG込み25万）
  - 検証ポイント: 田島さん層が「社外CFO月10万 + 運用代行月25万 = 35万」のうちレポート部分をいくらで切り出せるか
- **想定利益率（粗利率）**: 初期構築費用回収後 **80%以上**（限界費用はAPI料金とGCP費用のみ。月顧客あたり3000円前後）
- **差別化の核（1行で）**: 楽天実運用13店舗の現場知見が組み込まれた、ECに特化した「示唆付き」レポート（汎用BIツールは数字を出すだけ、汎用レポート受託は人力で遅い）

### コピー元候補（暫定）

- **HEARTCOUNT BI（韓国発、日本展開中）** / 対象: 中小〜中堅企業 / 価格: 月3万〜10万円 / 「これと何が違うか」: HEARTCOUNTは汎用BIで、ECの示唆ロジックは自前で組まないと出ない。当方はEC特化＋示唆込み
- **ノバセル（テレビCM効果分析、ラクスル系）** / 対象: 広告主企業 / 価格: 月数十万〜 / 「これと何が違うか」: ノバセルは広告効果に特化、当方はEC全体の経営KPI統合
- **Mazrica BI / Domo（中堅企業向けダッシュボード）** / 対象: 売上数十億〜 / 価格: 月10万〜数十万 / 「これと何が違うか」: 中小ECは対象外。設定コンサルも別料金。当方は中小EC特化で初期構築込み
- **コマースメディア／いつも.など楽天運用代行** / 対象: EC事業者 / 価格: 月20〜60万 / 「これと何が違うか」: 運用込みで人力依存・属人的。当方はレポート/分析に特化し、運用代行とは共存可能（バッティングしない）

**検証ポイント**: HEARTCOUNT / Mazrica BI の実際の中小EC導入率と離脱率。楽天運用代行各社のレポート品質に対する顧客満足度。

### 初期チャネル（最初の10社獲得想定）

- **チャネル1**: BPG経由の既存ネットワーク（13店舗の運営者・関係者経由で紹介）→ 最初の2〜3社はここで埋まる可能性大
- **チャネル2**: X（旧Twitter）のEC経営者クラスタへのコンテンツ配信（楽天RPP改善・月次レポート構築の知見をテンプレ化して発信、note記事を月2本）
- **チャネル3**: 楽天ECコンサル各社／運用代行会社経由の紹介（運用は彼ら、レポートは当方、というすみ分け提案）
- **チャネル4**: 既存の「コンテンツ配信システム（X自動投稿・note）」を実績デモとして活用 → 自社の自動化を見せること自体がリード獲得

### NGゾーン（このパターンで絶対やらないこと）

- 初年度はSaaS化しない（テンプレ化された受託型で顧客理解を深める。SaaS化は10社突破後）
- 楽天以外のチャネル（Amazon / Shopify）は「データ取得のみ対応」とし、運用代行は引き受けない
- 経営判断そのものの代行（コンサル）は本サービスに含めない。あくまで「データと示唆の提供」まで
- 顧客社数が15社を超えるまでは新機能追加せず、既存パイプラインの再現性向上に集中

## パターン2: K氏型 × A主軸「AIネイティブPMI業務統合」

### ターゲット顧客像（1人の架空ペルソナ）

- **名前（仮）**: 倉持 拓也さん（実在K氏のプロファイルから抽出）
- **年齢・役職**: 38歳・代表取締役（連続起業家、3社目を経営中）
- **会社規模**: ホールディングス売上25億円 / グループ従業員22名 / 設立4年 / 過去2年で5社買収
- **業種・商材**: 中小サービス業（地域工務店・整体院・税理士事務所等）のロールアップ。各社の「人に依存した業務」をAIで標準化し、グループとして上場を目指す
- **月次の経営課題（3つ、具体的に）**:
  1. **重複SaaSの統廃合地獄**: 買収した5社が会計（freee/MF/弥生で3つに分裂）、勤怠（KING OF TIME/ジョブカン）、CRM（Salesforce/HubSpot/Excel）でバラバラ。統合プロジェクトを内製でやろうとして3ヶ月止まっている
  2. **PMI後のKPI統一**: 買収各社の「売上」「粗利」「LTV」の定義が違い、HD全体の連結KPIダッシュボードが作れない。月次の経営会議で各社代表が「うちの数字はこう見て欲しい」と主張し続け、決着しない
  3. **業務マニュアル化と引き継ぎの属人性**: 買収先の創業者が辞めると業務が止まる。引き継ぎ用マニュアル作成を外注すると1社200万、3ヶ月かかる。AIで早く・安くやりたいが、誰もやり方を知らない
- **いま使っているツール／外注先（月額付き）**:
  - 各社個別のSaaS群（合計月50万以上）
  - 経営共創基盤系のPMIコンサル（スポット案件で1社500万〜1000万）
  - 業務マニュアル作成代行（1社200万、3ヶ月）
  - 内製の「業務統合PJ」担当者（年収700万、専任1名）
- **なぜこの人が「自動化」に金を払うか（1行）**: PMI支援を従来のコンサル価格で全買収先に投入していたら、ロールアップの利益が消えるから。「AIで10分の1の価格」を本気で探している

### サービス内容（最初に売る1プロダクト）

- **商品名（仮）**: 「PMI Bridge」（買収後企業のデータ統合・KPI統一・業務マニュアル化を3点セットで提供）
- **何をする**:
  1. 買収先のSaaSデータ（会計/勤怠/CRM）をAPI連携でHD側のBigQueryに統合
  2. KPI定義の擦り合わせワークショップ（HD CEO + 買収先代表 + 当方ファシリ）→ 統一KPIダッシュボード構築
  3. 買収先の主要業務（営業/受注/CS）の業務マニュアルをヒアリング→AIで自動生成→運用Notion化
- **顧客の作業（オンボーディング）**: 買収先1社あたり初回6時間ワークショップ + APIキー連携 + 業務ヒアリング3名×2時間 = 約2週間で初期構築完了
- **納品物**:
  - 統合KPIダッシュボード（HD用 + 買収先別画面）
  - 業務マニュアルNotion（買収先1社あたり主要10業務×SOP）
  - 月次PMI進捗レポート（経営会議用）
- **月額単価（仮置き）**: **初期構築 1社あたり80万〜150万円**（買収先1社につき） + **月次運用 1社あたり月10万円**
  - 検証ポイント: 倉持さん層が「コンサル500〜1000万」の代替として、初期80〜150万を払うか。複数買収先まとめ買いで割引すれば月収益が積み上がる
- **想定利益率**: 初期構築 50%（ワークショップ工数あり）/ 月次運用 75%
- **差別化の核（1行で）**: PMIコンサルが2-3ヶ月かけて作る「人力レポート」を、AI+標準テンプレで2週間・1/5の価格で提供

### コピー元候補（暫定）

- **山田コンサルティンググループ / 経営共創基盤 / 山田ビジネスコンサルティング** / 対象: PE/事業会社のPMI / 価格: スポット500万〜数千万 / 「これと何が違うか」: 人力コンサル、KPI統一に2-3ヶ月。当方はAI標準テンプレで2週間
- **PEファンド傘下のCoE（Center of Excellence）チーム例**: J-STAR、アドバンテッジパートナーズ等のポートフォリオ支援 / 内製のため外販なし / 「これと何が違うか」: 自社ファンド限定。当方は独立連続起業家向けに外販
- **Lightster / OneStream / Fluxer** などM&A後の財務統合SaaS / 対象: 米国中堅企業 / 価格: 年契約数百万円〜 / 「これと何が違うか」: 財務連結に特化、KPIと業務マニュアル統合はカバーしない。当方は3点セット
- **業務改善コンサル系（船井総研・タナベ等）** / 対象: 中小企業 / 価格: 月50万〜 / 「これと何が違うか」: PMI/ロールアップ特化ではない、AIネイティブでもない

**致命的検証ポイント**: 日本でM&Aロールアップを実行している連続起業家は推定数百〜千社。**市場規模が小さい**。コピー元の中でも「日本の中小ロールアップ向けに3年以上続いて儲かっている事業」が**ほぼ存在しない可能性**。Koerner原則「コピーファースト」の最大の警告対象。

### 初期チャネル（最初の10社獲得想定）

- **チャネル1**: K氏（倉持さんの実在モデル）との直接接触 → 第1顧客化＋紹介ハブ化

- **チャネル3**: M&Aアドバイザリー会社経由の紹介（M&A総研、日本M&Aセンター、ストライク等のディール後フェーズへの食い込み）

### NGゾーン

## パターン3: AB融合「EC月次経営オペ一式」パッケージ

### ターゲット顧客像（1人の架空ペルソナ）

- **名前（仮）**: 大澤 美咲さん
- **年齢・役職**: 36歳・代表取締役（自身もプレイヤー的にEC運用を見ている）
- **会社規模**: 売上3.5億円 / 従業員8名（うちEC運用4名）/ 設立5年
- **業種・商材**: アパレル/雑貨の自社ブランド、楽天主軸（楽天月商700万、自社EC比率20%）
- **実在クライアントの近似像**: **Her lip to BEAUTY / GINZA-HANACHA に近い**（自社ブランド主導、楽天運用が経営の中核）
- **月次の経営課題（3つ、具体的に）**:
  1. **データ更新の社内地獄**: 楽天RMSへの価格改定・在庫同期・新商品登録・レビュー集計を、運用担当2名が毎日合計4時間費やしている。月80時間×時給3000円＝月24万の人件費が「機械的作業」に消えている
  2. **広告運用と分析の分離**: 楽天RPPは外注（月20万）、Amazon広告は内製、Shopify広告は別の外注。統合的なROI分析ができておらず、広告予算配分が毎月「気合と前月の数字を見て」決まる
  3. **月次レポートと意思決定の断絶**: 月初に「数字をまとめたPDF」は来るが、経営判断（どの商品を仕入れ強化するか、どの店舗を閉じるか）に使える示唆がない。結局自分で楽天RMSとSlack履歴を見直して判断している
- **いま使っているツール／外注先（月額付き）**:
  - 楽天RPP運用代行（月20万）
  - 業務委託のEC運用補助スタッフ（月15万、データ更新業務含む）
  - 月次レポート作成（社内運用担当が手作業、工数換算で月8万相当）
  - 楽天RMS / Amazon SC / Shopify / Slack / Notion / freee
- **なぜこの人が「自動化」に金を払うか（1行）**: 自分自身がプレイヤー兼経営者で時間が足りず、「機械的作業を全部巻き取ってくれる外部パートナー」を心底求めているから

### サービス内容（最初に売る1プロダクト）

- **商品名（仮）**: 「EC経営オペレーション・フルパッケージ」
- **何をする**: 以下を一式で提供する月次オペ代行：
  1. **月次経営レポート**（パターン1と同じ）+ 週次Slack通知 + ダッシュボード
  2. **データ更新代行**: 価格改定・在庫同期・新商品登録・レビュー集計の自動化＋例外手動対応
  3. **広告レポート統合**: 楽天RPP / Amazon広告 / Shopify広告 の統合ROI分析（運用代行とは別、分析・示唆のみ）
- **顧客の作業（オンボーディング）**: 初回2時間ヒアリング + APIキー連携 + 現行業務の棚卸し（運用担当との3時間セッション）。所要2〜3週間
- **納品物**:
  - パターン1の納品物 全て
  - + データ更新自動化スクリプト（顧客環境にデプロイ or 当方環境でAPI連携）
  - + 統合広告レポート（週次・月次）
  - + 月2回30分のオペレーション読み合わせMTG
- **月額単価（仮置き）**: **月30万円〜50万円**（基本30万 / 広告統合・週次MTG込み50万）
  - 検証ポイント: 大澤さん層が「現状の運用補助月15万 + レポート社内工数月8万 + 広告外注月20万」のうち、どこまで巻き取れるか
- **想定利益率**: **40〜55%**（データ更新代行のオペレーション工数が嵩む。完全自動化できる範囲を見極めるまで属人化リスク）
- **差別化の核（1行で）**: EC運用代行（人力）でもなく、BIツール（数字のみ）でもない「経営オペレーションの自動化＋例外対応パートナー」

### コピー元候補（暫定）

- **コマースメディア / いつも. / SAVAWAY / EC-CUBE運用代行各社** / 対象: EC事業者 / 価格: 月20〜80万 / 「これと何が違うか」: 人力運用代行、レポートは付属。当方は自動化・データ・示唆が主、運用は最小限
- **EC-Force / リテリア等 EC運用支援SaaS** / 対象: D2Cブランド / 価格: 月数万〜十数万 / 「これと何が違うか」: ツール提供のみ、オペ代行なし。当方は「ツール＋オペ」のハイブリッド
- **コマースルーム / Hamee運用代行系** / 対象: 中小EC事業者 / 価格: 月15〜40万 / 「これと何が違うか」: 楽天運用に特化、データ統合・経営レポートは弱い

**致命的検証ポイント**: 「データ更新代行」は属人化しやすく、限界費用が高い（顧客あたりに人手が必要）。Koerner原則「利益が真実」と、ユーザーのスターター気質（運用フェーズでの退屈離脱リスク）の二重リスク。

### 初期チャネル（最初の10社獲得想定）

- **チャネル1**: BPG既存13店舗の関係者（パターン1と同じ）
- **チャネル2**: 楽天運用代行会社経由の紹介（運用＝彼ら、自動化＝当方）
- **チャネル3**: X / note のEC経営者向けコンテンツ
- **チャネル4**: D2Cブランドコミュニティ（Cult、Shopify Japan系イベント等）

### NGゾーン

- 1社あたりのデータ更新業務に月20時間以上の人手が必要な顧客は受けない（属人化・利益率崩壊）
- 楽天RPP運用そのものは受けない（運用代行各社と競合しない領域に留まる）
- 顧客の「経営判断」自体の代行は受けない（コンサル領域に踏み込まない）
