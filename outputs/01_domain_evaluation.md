# ドメイン評価マトリクス（Day 1 / Phase 1）

## 評価基準

| 軸 | 説明 | 配点 |
|----|------|------|
| 市場規模 | ターゲット企業数 × 支払い意思 | /5 |
| 実装容易性 | 既存の実績・スキルからの距離 | /5 |

| 競合の薄さ | 既存プレイヤーの強さ・数 | /5 |

| コピー元の有無 | 既存成功モデルが存在するか（Koerner原則） | /5 |

## 候補別詳細分析

### 1. 業務プロセス自動化 — 合計24点 → GO候補

**なぜスジが良いか:**
- BPGで実証済み: 20エージェント・43スキル・日次パイプラインを6週間で構築した実績がそのまま武器になる
- 「AIエージェント構築」「社内オペレーション自動化」の両方をカバーできる
- 中小企業の「人を雇うほどではないが手作業では回らない」領域は広大

**コピー元（成功モデル）:**
1. Zapier / Make — ノーコード自動化（ただしAIエージェントレイヤーは薄い）
2. AI受託開発会社（日本: SHIFT AI、海外: AI implementation agencies）
3. 業務コンサル+実装のハイブリッド（アクセンチュア型を小規模に）

**最大リスク:**
- 「業務プロセス自動化」は範囲が広すぎる。「誰の・どの業務を」を絞らないと提案が刺さらない
- 受託は工数商売。月利を上げるには、繰り返し使えるテンプレート化が必須

**競合が厚い理由（2点）:**
- 大手SIer、フリーランスエンジニア、ノーコードツールが全て競合になる
- ただし「AIエージェントを業務に実装できる人材」はまだ希少。ここが差別化ポイント

**市場規模（4点の根拠）:**
- 日本の中小企業約360万社（根拠: 中小企業庁）
- そのうちDX投資意向がある企業は約30%（根拠なし、推定）
- 単価30万〜100万円/案件として、TAMは大きいが到達可能市場は狭い

---

### 2. データ集計・レポート自動化 — 合計20点 → 保留

**強み:**
- 月次スライドレポート自動生成パイプラインが完成形として存在する
- BPGの13店舗で実証済み

**弱み:**
- 候補1の部分集合にすぎない。単体で事業にする必要があるか疑問
- ユーザー自身が「同じ作業の繰り返しがつまらない」と回答。レポート案件は本質的にルーティンになる
- 熱量3点 — 1年続けるには低い

**判定: 保留** — 候補1のサブサービスとして組み込む方が自然

---

### 3. 社内ナレッジ・情報検索 — 合計15点 → KILL

**KILL理由:**
- 競合が極めて厚い（Notion AI、Glean、Guru、Slite、各社RAGソリューション）
- ユーザーの実績にこの領域の直接的な成果物がない
- 差別化が見えない
- 熱量2点 — 本人がやりたいと思っていない

---

### 4. 外注業務のAI内製化 — 合計18点 → KILL

**KILL理由:**
- コピー元が弱い（2点）。成功している類似ビジネスが少なく、市場が未検証
- 「クリエイティブ・広告運用のAI化」は技術変化が激しく、提供価値の賞味期限が短い
- BPGでのコンテンツ配信システム実績はあるが、外販するには顧客の課題定義が曖昧
- 熱量3点 — 最優先ではない

---

## Phase 1 暫定結論

**GO候補: 業務プロセス自動化（AIエージェント構築 + 社内オペレーション自動化）**

次のステップ（Day 2-7で実施）:
1. 「誰の・どの業務を」のターゲット絞り込み（顧客マッピング）
2. コピー元3社の価格体系・プロセス・チャネル調査
3. Koerner式コピー元調査: 日本で「AIエージェント構築を受託している」会社を10社リストアップ

## 起業家タイプ診断

| 項目 | 結果 |
|------|------|
| タイプ | **スターター×メインテナー** |
| 完成後の傾向 | 改善し続けたい（メインテナー） |
| 退屈の正体 | 同じ作業の繰り返し（スターター） |
| 営業適性 | 苦手だがやる覚悟あり |
| 補完パートナー | 将来的に営業/BD担当が必要。初期は自力で泥臭くやる |
| 退屈サイクル | 推定3-6ヶ月。同じ案件タイプが続くと退屈する可能性大 |

**対策:**
- 受託の型化（テンプレート）を進め、ルーティン部分を仕組みに落とす
- 案件ごとに「新しい技術要素」を1つ入れることで退屈を防ぐ
- スケール後はオペレーション担当を採用し、自分は新規案件設計に集中する構造を目指す
