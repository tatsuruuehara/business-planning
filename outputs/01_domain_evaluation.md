## 6軸スコアリング（各5点満点 / 合計30点）

| 軸 | A: 業務プロセス自動化（EC×少数精鋭） | B: データ集計レポート自動化 |
|---|---|---|
| **1. 市場規模** | **3** / EC事業者は数万社規模だが「5〜10人で自動化に投資する層」は数千社オーダー。Q6のK氏のような層は更に少ない（数百社） | **4** / レポート需要はEC外に広く、中小企業全体（数十万社）に潜在。実証ドメインがECなだけで横展開余地が大きい |
| **2. 実装容易性**（既存スキル・実績からの距離） | **3** / 「業務」が未確定なため、案件ごとに作り込みが要る可能性。受託寄りで再利用率が下がる懸念 | **5** / Sheets→Claude→Slides→Slack の完全パイプラインが既に稼働。13店舗運用実績あり。**明日からテンプレ販売可能** |
| **3. 本人の熱量** | **5** / Day 1で「熱量5」を自己申告。経営者の抽象アイデアを実装に落とす作業＝最も得意かつ好む領域 | **3** / 実証済みだが「熱量未確認」。レポートだけだと作業感が強く、スターター気質と相性が悪い可能性 |
| **4. 競合の薄さ** | **2** / 広告自動化ツール（月38万払う対象）は既に大手プレイヤーがいる。業務自動化系の受託も多数（Make, Zapier, n8nの代理店化など） | **3** / BIツール（Looker Studio, Domo, Tableau）+ レポート受託の複合戦場。ただし「AI×経営レポート×ゼロタッチ」の組み合わせは薄い |
| **5. 課金ポテンシャル**（月利ベース） | **2** / 月6万のデータ更新を10社で月60万売上→月利30万未満。月38万の広告ツール置換は単価大だが受注確度低。**スケール時の天井が見えない** | **4** / 月10〜30万でレポートサブスク × 30社 = 月300〜900万売上 / 月利200〜600万。**実証済みパイプラインで限界費用が低い** |
| **6. コピー元の有無** | **2** / Q7で具体名なし。「業務自動化系受託」は無数にあるが、3年以上続いて儲かっている「中小EC向け業務自動化SaaS/サービス」の固有名詞が出ない＝市場未検証 | **3** / レポート自動化系（HEARTCOUNT、Spider AF的BI/可視化、Mazrica、ノバセル等）は存在。経営ダッシュボード受託会社も多数。コピー元探索は容易 |
| **合計** | **17 / 30** | **22 / 30** |

- 18〜22点: GO候補（仮説確立フェーズに進める）
- 23点以上: 強GO候補（優先投資）

→ **A は 17 点で閾値割れ。B は 22 点で GO候補**

### ギャップが教えるもの

- **A の致命傷は「課金ポテンシャル」「競合の薄さ」「コピー元」の3軸が全て2点**。熱量5だけで突っ込むと、Koerner原則「利益が真実」「コピーファースト」を二重に踏み外す
- **B の弱点は「熱量3」のみ**。これは「レポート単体だと作業感が強い」という構造的問題で、後述パターン2・3で補える
- 重要: **A と B はトレードオフではなく、A の弱点（課金・コピー）を B の強み（実証済み・限界費用低）で補完できる可能性がある**

## 暫定推奨

**ストレートな B 単独ではなく、B を主軸にしつつ A の熱量領域を周辺に組み合わせるハイブリッド構成を推奨する。**

理由3点:
1. 数字（22点 vs 17点）が示す通り、独立採算の事業として B が圧倒的に堅い
2. ただし B 単独だと熱量3で1年続かない可能性（スターター気質との不整合）
3. A の「経営者の抽象アイデアを実装に落とす」熱量は、B の上に**コンサル/拡張サービスとして乗せる**ことで活かせる

## 3つの絞り込みパターン（次の意思決定対象）

ユーザーは以下3つから Day 3 までに1つを選ぶ。

### パターン1: 「EC実証強み主導」B主軸 × A抱き合わせ

**コンセプト**: EC事業者向け「月次経営レポート自動生成 SaaS / サービス」を本丸にし、データ更新・在庫同期などの A 系業務を周辺オプションで売る

- 強み: 13店舗実証済み・明日からテンプレ販売可能・最短で課金開始
- リスク: ターゲットを EC に縛ると市場規模天井（数千社）が早く来る。熱量3で1年続くか
- 明日の一手: BPG既存13店舗のレポートを「外販可能なパッケージ」に切り出し、価格表を作る

### パターン2: 「K氏型主顧客 × 新規市場開拓」A主軸

**コンセプト**: AIネイティブな少数精鋭組織（特にM&Aロールアップ志向の連続起業家）を主顧客にし、買収後企業の業務統合・自動化を売る

- 強み: 熱量5・K氏という実在の顔がいる・市場が未開拓で先行者利益あり
- リスク: 顧客数が極小（日本で数百社以下）・営業サイクルが長い・コピー元なし＝市場未検証
- 明日の一手: K氏と1時間の壁打ちアポを取り、買収後企業で実際に発生している「業務統合の地獄」を3つ言わせる

### パターン3: 「EC月次経営オペ一式」AB融合パッケージ

**コンセプト**: EC事業者向けに「月次レポート + データ更新 + 広告ツール統合」を一式パッケージで提供。月額20〜40万円の経営オペ代行

- 強み: 単価が積み上がる（B単独より3〜5倍）・実証済みパイプラインを軸にできる・熱量と実装容易性の両立
- リスク: 受託寄りで属人化リスク・サービス境界が広く工数膨張の懸念・既存ECコンサル会社との競合
- 明日の一手: BPG13店舗で実際に発生している月次オペ業務を全リスト化し、パッケージ化可能な範囲を見極める

---

## 暫定推奨パターン

**パターン1（B主軸 × A抱き合わせ）を推奨する。** ただし Day 3 にコピー元調査結果と K氏接触可否で見直しする。

- パターン2 は熱量は最高だが、コピー元なし＋市場規模未確定＝**Koerner原則で言う「市場未検証リスク」が最大**。サイドビジネスとして検証する余裕がない
- パターン3 は魅力的だが、Day 1 の起業家タイプ診断「スターター」と相性が悪い。受託サービスは運用フェーズが長く、退屈で離脱する可能性が高い
- パターン1 は「実証済みを最速で外販→キャッシュフローを作る→そこから A 系を選択的に追加」という、Koerner式の段階拡張に最も合う
