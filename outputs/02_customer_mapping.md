# 顧客マッピング（Day 1 / Phase 1）

## ポジショニング

**「AIエージェント構築による業務自動化」の受託サービス**
- ECに特化しない。ただしEC事業者を最初の顧客として獲得する
- 提供価値の核心: 「経営者の頭の中にある業務フロー → AIエージェントで自動化」

## 顧客セグメント

### Tier 1: 今すぐ売れる（引き合いあり）

| 属性 | 詳細 |
|------|------|
| 業種 | EC事業者（楽天・Amazon出店） |
| 規模 | 月商500万〜3,000万円 |
| 役職 | 代表 or EC事業部長 |
| 課題 | レポート作成に月1-2日、運用が属人化、データはあるが活用できない |
| 現在の支出 | EC運用代行 月15-50万円 / BIツール 月3-10万円 / 外注レポート 10-30万円/回 |
| 自社の提供 | レポート自動生成パイプライン + 運用ダッシュボード（EC運用代行はやらない） |

**具体リーチ先:**
- 引き合いが来ているEC事業者（1-3社）
- BPGの経営者からの紹介可能性

### Tier 2: 近い将来売れる（EC周辺）

| 属性 | 詳細 |
|------|------|
| 業種 | EC支援会社、広告代理店、物流会社 |
| 規模 | 社員10-50人 |
| 役職 | 代表 or 事業部長 |
| 課題 | クライアント向けレポート作成が手作業、社内ナレッジが散逸、定型業務が多い |
| 現在の支出 | 人件費（レポート作成担当者の工数） |
| 自社の提供 | AIエージェントによる社内業務自動化（レポート、データ集約、アクション管理） |

### Tier 3: 将来の拡張市場（非EC）

| 属性 | 詳細 |
|------|------|
| 業種 | 不動産、飲食チェーン、士業、人材 |
| 規模 | 社員5-30人 |
| 役職 | 代表 |
| 課題 | 業務が属人化、レポート・集計が手作業、AIを使いたいが何から始めればいいかわからない |
| 現在の支出 | 外注（コンサル・開発） or 何もしていない |
| 自社の提供 | 業務プロセス診断 → AIエージェント設計 → 構築 → 運用支援 |

## Koerner式コピー元調査

### 日本で類似サービスを提供している企業

| 企業/個人 | 提供内容 | 価格帯 | 備考 |
|-----------|----------|--------|------|
| SHIFT AI | AI導入コンサル + 実装支援 | 月50-200万円（推定） | 大企業向け。中小は対象外 |
| AIコンサル系フリーランス | ChatGPT/Claude活用支援 | 月10-50万円 | 「使い方教える」止まりが多い |
| EC運用代行会社 | EC全般の運用代行 | 月15-50万円 | AIエージェントは組み込んでいない |
| Zapier/Make代理店 | ノーコード自動化 | 月5-20万円 | AIエージェントレイヤーが薄い |

### 差別化ポイント

1. **AIエージェントを「作る」ではなく「業務に実装する」**: 技術デモではなく、業務フローに組み込んで月次で回る仕組みを納品する
2. **実証済みの本番稼働実績**: 13店舗の日次パイプライン、ゼロタッチ月次レポートは「動いている証拠」
3. **経営者の言語を翻訳できる**: 抽象的な「こんなことできない？」を実装可能な仕様に落とせる。これはエンジニアにもコンサルにもできない中間領域

## 初期ターゲット（Phase 2で検証する顧客）

1. 引き合いが来ているEC事業者 1-3社 → **最優先。今週中にヒアリング設定**
2. BPG経営者への紹介依頼 → 1-2社
3. X経由での接点構築 → 並行で発信開始

## 価格仮説（検証前の初期設定）

| メニュー | 価格 | 内容 |
|----------|------|------|
| 業務診断 + 設計 | 30-50万円（初期） | 業務フロー可視化 → AIエージェント設計書 |
| 構築・実装 | 50-150万円（案件規模次第） | エージェント構築 → テスト → 本番投入 |
| 運用保守 | 月10-30万円 | 月次改善 + 障害対応 + 追加開発 |

**Koerner原則に基づく注意:**
- この価格は仮説。検証前に固定しない
- まずTier 1の1-3社にヒアリングし、「いくらなら払うか」を直接聞く
- コピー元の価格体系を参考にしつつ、利益ベースで判断する
