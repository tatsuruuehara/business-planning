# 事業コンセプトシート v1

## 1. 一行コピー

**中小EC事業者の属人業務をAIエージェントで自動化し、月次で回る仕組みに変える**

## 2. ターゲット顧客（具体）

| 項目 | 内容 |
|------|------|
| 業種 | EC事業者（楽天・Amazon出店）→ 実績後にEC支援会社・広告代理店に拡張 |
| 規模 | 月商500万〜3,000万円 / 社員3-20人 |
| 役職 | 代表 or EC事業部長 |
| いま何に困っている | レポート作成に月1-2日、運用が属人化、データはあるが活用できない、人を雇うほどではないが手作業では回らない |
| いま何にいくら払っている | EC運用代行 月15-50万円 / BIツール 月3-10万円 / 外注レポート 10-30万円/回 / 競合調査 数十万円/回 |

## 3. 提供価値（Before / After）

| | 状態 |
|---|---|
| **Before** | 月次レポートに半日〜1日、広告データは手動でダウンロード、競合調査は外注か未実施、施策検討は勘と経験 |
| **After** | レポートは毎月自動生成されSlackに届く、ダッシュボードで日次KPIが見える、競合データが自動収集され差分がわかる |
| **削減/創出** | 月20-40時間の工数削減 ≒ 月10-20万円相当 / 意思決定スピード向上（定量化困難） |

## 4. プロダクト概要

| 項目 | 内容 |
|------|------|
| 形態 | **受託（伴走型）+ 月額保守** → 将来的にテンプレSaaS化を検討 |
| 主要機能 | ①月次レポート自動生成（Sheets→Claude→Slides→Slack） ②日次KPIダッシュボード（Cloud Functions→GitHub Pages） ③競合リサーチ自動化（API→分析→レポート） |
| AIの使いどころ | Claude APIによるデータ分析・文章生成・施策提案 / マルチエージェント協調（分析→生成→QA） |

## 7. 競合と差別化

### 既存代替

| 代替手段 | 弱点 |
|----------|------|
| EC運用代行会社 | AIエージェント未実装。人力運用の延長 |
| BIツール（Tableau等） | ダッシュボードは作れるがレポート自動生成・施策提案はない |
| AIコンサル系フリーランス | 「使い方教える」止まり。本番稼働する仕組みを納品しない |
| Zapier/Make代理店 | ノーコード自動化は得意だがAIエージェント層が薄い |
| 大型AI受託（WEEL等） | PoC 150万〜。中小EC事業者には高すぎる |

### 自分の差別化（3つ）

1. **「動いている証拠」がある**: 13店舗の日次パイプライン、ゼロタッチ月次レポート、20エージェント・43スキルが本番稼働中。デモではなく実績
2. **EC業務の言語で話せる**: RPP、CVR、ROAS、SKU — エンジニアには話せない、コンサルには作れない中間領域を埋める
3. **圧倒的に安い**: 大型受託の1/10〜1/5。中小ECが手を出せる価格帯で、本番稼働するものを納品する

## 8. なぜ自分がやるか

| 項目 | 内容 |
|------|------|
| 強み接続 | BPGで6週間かけて構築したエージェントシステムがそのまま商品の原型。ゼロから作るのではなく、実績をパッケージ化する |
| 熱量スコア | **4/5**（AIエージェント構築自体が楽しい。ただし同じ案件の繰り返しは退屈する→テンプレ化で対策） |
| 1年やり続けられる理由 | 案件ごとに新しい業務ドメインに触れられる。構築→改善のサイクルがスターター×メインテナー気質に合う |

## 9. 最大リスクと撤退基準

| 項目 | 内容 |
|------|------|
| 最大リスク | **「AIエージェント構築」の需要が中小EC事業者に実在しない可能性**。「面白そう」と言われるが財布が開かないパターン |
| 撤退基準 | 3ヶ月以内に有償案件（Menu A以上）が1件も取れなければ KILL。ピボット先: EC特化のSaaSダッシュボード（月額制） |
| 二番目のリスク | 本業（BPG）との時間配分。週半分のリソースで月2-3件を回せるかの工数検証が必要 |
